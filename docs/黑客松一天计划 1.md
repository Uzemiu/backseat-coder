# 黑客松一天计划

只有一天。这份计划的核心不是"做多少"，而是**保证任何时刻手上都有一个能演示的东西**，然后在还有余力时逐层加亮点。

原则：[实现架构.md](实现架构.md) 已论证——大脑层（Node 服务）现成，插件只做"感知 + 呈现"。所以一天的活全在插件侧。

---

## 一、最重要的一句话

> **先保证有能跑的 demo，再追求愿景完整。**

每个阶段结束都要有一个"如果现在必须演示，我能演示什么"的答案。绝不把所有时间押在最难的功能上。

---

## 二、分层目标（MoSCoW）

| 层级 | 目标 | 说明 |
|------|------|------|
| **Must（保命）** | 侧边栏插件能跑，能扫仓库、能看 diff 审视 | 把现有能力搬进 VS Code 侧边栏。做到这步 = demo 已经成立 |
| **Should（加分）** | 编辑感知 + AFK 静默 | 能演示"它知道你在敲、知道你走了"——这是愿景的核心差异点 |
| **Could（亮点）** | 犹豫感知卡片 | 有余力才做，调参不准也无伤大雅 |
| **Won't（今天不做）** | 正文内浮现、风格记忆磨合、插件自动拉起服务、JetBrains | 明确不碰，避免时间黑洞 |

## 三、按小时排期（约 8-9 小时工作日）

### 🟢 上午：保命层（目标——中午前有能演示的插件）

**09:00–10:00｜环境 + 骨架**
- 确认本地服务能跑：`npm run dev`，浏览器验证 `/api/scan` 有响应
- `yo code` 生成 TS 插件，`package.json` 声明侧边栏容器 + WebviewView
- F5 能启动扩展开发宿主，侧边栏出现一个空面板
- ✅ 检查点：侧边栏里能看到一个"能工智人"的空面板

**10:00–11:00｜打通一条 fetch 链路**
- 插件里 `fetch('http://localhost:3000/api/scan', ...)`，把结果 `console.log` 出来
- 解决 CORS / 网络问题（本地对本地一般没事）
- ✅ 检查点：点一下按钮，控制台能打出真实的 repoMap

**11:00–12:30｜侧边栏卡片流 + 展示项目理解和 diff**
- Webview HTML/JS：一个能往下追加卡片的容器
- 插件 ↔ webview 的 `postMessage` 协议打通（最简版）
- 接入 `/api/scan`（项目理解卡片）和 `/api/diff`（改动审视卡片，先做手动按钮触发）
- ✅ **保命检查点：点"扫描"出项目理解卡片，点"看改动"出 diff 审视卡片。到这步，demo 已经能讲完整故事了。**

> ⏰ 如果到 12:30 还没到这个检查点，停止一切加分项，全力保这条线。

### 🍚 午休

### 🟡 下午：加分层（目标——演示"它感知你的状态"）

**13:30–15:00｜编辑行为感知 + 自动 diff 审视**
- `onDidSaveTextDocument` / `onDidChangeTextDocument`，加防抖（关键，先做）
- 保存文件 → 自动调 `/api/diff/check`（带 `previousDiffHash`，不变不烧 token）
- diff 变了 → 自动往侧边栏加一张卡片
- ✅ 检查点：改代码保存后，侧边栏自动出现审视卡片，不用手点

**15:00–16:30｜AFK 在场检测**
- 记录最后活动时间；定时器判断空闲超阈值 → 静默
- `onDidChangeWindowState` 失焦也静默
- 静默时停止轮询、显示"💤 已静默"卡片；恢复活动唤醒
- ✅ 检查点：停手 2 分钟，侧边栏显示已静默；回来敲键，恢复。**这是最能打的演示点——省 token + 懂分寸肉眼可见。**

### 🔴 傍晚：亮点层 / 打磨（视剩余时间）

**16:30–17:30｜二选一**
- **选项 A（有余力）**：犹豫感知——反复删改同一段 → 加一张"这里改了好几遍，需要帮忙理清吗"卡片。调参不准就调宽阈值，宁可少报
- **选项 B（更稳）**：打磨已有的——卡片样式、AI 来源标识（local/anthropic）、任务导航接入 `/api/navigate`

**17:30–18:30｜Demo 准备（雷打不动留出来）**
- 准备一个改动明显的目标仓库
- 走一遍 demo 脚本，确认每步都稳
- 录一段屏做备份（现场翻车有兜底）

## 四、Demo 脚本（演示时照着走）

```
1. 打开 VS Code，侧边栏已有"能工智人"面板
2. 点"扫描" → 侧边栏出现项目理解卡片（这是什么项目、核心模块、从哪读）
   台词：它不写代码，先帮我读懂这个项目
3. 改一处代码并保存 → 侧边栏自动冒出改动审视卡片（影响、风险、缺的测试、反思问题）
   台词：我写，它看。它不改我的代码，只提醒我想清楚
4. 停手两分钟（或切走窗口）→ 侧边栏显示"💤 已静默"
   台词：我去接杯咖啡，它就安静了，不鞭策我、不烧 token
5. 回来敲键 → 唤醒
   台词：我回来，它也回来
6.（若做了犹豫感知）反复改同一段 → "这里改了好几遍，要帮忙吗"
```

**Demo 一句话主张**：市面上的 AI 抢着帮你写代码；能工智人反过来——**你写，它在旁边看，看你懂没懂、卡没卡、在不在。**
